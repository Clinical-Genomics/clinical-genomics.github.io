Beställningar

Beställningar inom pågående projekt görs via vår portal på www.clinicalgenomics.se.

För att diskutera nya projekt, kontakta Valtteri Wirta.

Skapa ny beställning

  1. Kontrollera att provet uppfyller provkraven (prover som inte uppfyller kraven kan skickas in, men analyskostnaden kommer att debiteras oavsett resultat)
  2. Fyll i beställningsformuläret Order Form
  3. Logga in på vårt webbaserade kommunikationsverktyg SupportSystems och skapa en ny ticket. Bifoga det ifyllda orderbladet.
  4. Skicka in proverna på valfritt sätt (se adresser nedan). Bifoga utskrift av beställningsformuläret samt notera det ticketnummer som erhålls via e-post

Veckoschema

Exomsekvensering/Low-input exomsekvensering (EXOSXTRxxx/EXOSLIRxxx) måndagar kl 10.00

Helgenomsekvensering (WGSPCFC030/60) onsdagar kl 08.00

Prioritering av prover

Standard - provet analyseras enligt veckoschemat.

Förtur/prioriterat - provet analyseras enligt veckoschemat, men provet får förtur vid varje möjligt tillfälle. Tex först till sekvensering och först till bioinformatisk analys. Ingen extra kostnad för detta, men förutsätter ‘fair usage’.

Express - provet analyseras snarast och utanför veckoschemat. Förväntad leveranstid ca 5 arbetsdagar. Extra kostnad 5000 kr. Tillgänglighet för denna analys bekräftas separat för varje prov.

Orders through the portal (BETA)

Sign up in the order portal

  1. Go to https://clinical-db.scilifelab.se:8080 and click “Login”
  2. Pick a Google account if you are presented with multiple options
  3. Select which customer(s) you belong to and click “Save”

Remember to use the same Google account when you log in later. You are now ready to submit samples!

Create a new order (project)

  1. Fill in a relevant “Project name”. Select which customer you want to submit the order as. Finally click “Add new project”.

  2. Each sample belongs to a family so start by adding one. An order can contain multiple families.
    • The “Family name” must be unique for the customer and can’t contain spaces
    • Mark “Existing family” if you want to extend a previous family with more samples (e.g. parents to a child)
  3. Add samples to the family.
    • Different fields are required depending on e.g. which “Application tag” is picked - the interface will inform you about missing information.
    • Mark “Existing” if you want to update some information for a previously sequenced samples (e.g. adding parents to a child)

  4. Review the information and submit the project by finally clicking “Submit project”

    All information is stored until you submit the project. You can come back later to update something and add more samples. As soon as you submit the project it will disappear from the interface and you are unable to make updates.

  5. The interface will create a ticket in SupportSystems where further communication will be handled. You should have received an email about the ticket which you can respond to if you have additional information about the order.

You can always update information by changing filled in values and clicking the corresponding “Update” button.
